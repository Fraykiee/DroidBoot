# DroidBoot

> Turn a rooted Android phone into a bootable USB drive - like DriveDroid.
> Телефон по USB-кабелю эмулирует флешку с твоим ISO/IMG, и ПК грузится прямо с него.

Android-приложение (требует **root**), которое превращает телефон в загрузочный
USB-накопитель для установки/запуска Windows или Linux. Удобно, когда под рукой нет
флешки и картридера, а поставить ОС надо.

## Как это работает

На рутованном Android ядро Linux умеет презентовать себя хосту как произвольное
USB-устройство через **configfs** (`/config/usb_gadget`). Приложение находит
системный USB-гаджет, подкладывает в `mass_storage` lun путь к образу и переключает
композицию USB - ПК видит «флешку» с твоим ISO.

Используются два метода (выбираются автоматически):

- **property-метод** - для устройств, где USB композится через `sys.usb.config`
  (большинство Qualcomm/MediaTek, Xiaomi и др.). Неразрушающий: не ломает штатный
  гаджет, переключение делает init по своим триггерам.
- **ручная привязка UDC** - fallback, когда системного гаджета/триггеров нет:
  поднимаем собственный гаджет и сами биндим его к UDC.

## Режимы прошивки целевого ПК

| Режим | Атрибуты lun | Когда выбирать |
|-------|--------------|----------------|
| **UEFI (CD-ROM)** | `cdrom=1`, `removable=1` | Самый совместимый. El Torito CD, грузится в UEFI и Legacy. |
| **BIOS (USB-HDD)** | `cdrom=0`, `removable=1` | isohybrid ISO / `.img` как обычная флешка. |
| **Старый BIOS (фикс. диск)** | `cdrom=0`, `removable=0` | Старые **Award/Phoenix** BIOS показывают носитель только если он притворяется *фиксированным* диском - ищи телефон в **Hard Disk Boot Priority**. |

## Совместимость устройств

Механизм упирается в **USB-контроллер** ядра. Приложение определяет его на экране
«Готовность устройства» и предупреждает заранее.

| Контроллер | SoC (примеры) | Статус |
|------------|---------------|--------|
| **DWC3 / DWC2** | Qualcomm Snapdragon и мн. др. | ✅ Работает. Проверено живьём на **Xiaomi Mi 8 Lite (SDM660)** - хост видит «Linux File-Stor Gadget», ядро стабильно. |
| **MUSB** | вендорные ядра **MediaTek** | ❌ Энумерация `mass_storage` роняет ядро (`musb_g_ep0_irq` NULL deref, `ep1in dma_mapping_error` → ребут в прелоадер). Баг вендорного ядра, из userspace **не чинится**. Проверено на Helio G100 (MT6789). |
| прочие | - | ⚠️ Не проверено - пробуй на свой риск. |

> На MUSB-устройствах приложение покажет красное предупреждение и не даст словить
> внезапный ребут вслепую.

## Требования

1. **Root** (Magisk / KernelSU).
2. Ядро с `CONFIG_USB_CONFIGFS_MASS_STORAGE` (есть почти везде - используется для
   MTP/ADB-композиций).
3. Доступный UDC в `/sys/class/udc` и USB-контроллер из «зелёного» списка выше.

## Сборка

Открой проект в **Android Studio** (Giraffe+) → Run. Или из консоли:

```bash
gradle wrapper          # один раз - сгенерировать gradlew (если его нет в репо)
./gradlew assembleDebug  # Windows: gradlew.bat assembleDebug
```

APK появится в `app/build/outputs/apk/debug/`.

- `applicationId` / `namespace`: `com.fraykiee.droidboot`
- `minSdk 29`, `targetSdk 34`, Kotlin + Jetpack Compose (Material 3)

## Использование

1. Дай приложению root.
2. Проверь экран «Готовность устройства» (root, configfs, UDC, контроллер).
3. Выбери образ (ISO/IMG) и режим прошивки.
4. Нажми «Подключить» - телефон станет USB-накопителем.
5. На ПК выбери телефон в загрузочном меню/BIOS.
6. «Отключить» вернёт обычный USB (ADB/MTP).

> ⚠️ Подключай телефон **до** включения ПК (или через перезагрузку), порт лучше
> задний **USB 2.0**. Образ открывается только для чтения (`ro=1`).

## Структура

| Файл | Назначение |
|------|-----------|
| `usb/UsbGadgetManager.kt` | Вся configfs-логика: поднять/опустить/сменить образ, классификация контроллера |
| `usb/PathResolver.kt`     | `content://` Uri → реальный путь для root |
| `usb/GadgetService.kt`    | Foreground-сервис, держит гаджет живым |
| `root/RootShell.kt`       | Обёртка над libsu |
| `model/BootImage.kt`      | Модель образа и `FirmwareMode` |
| `MainViewModel.kt`        | Состояние UI |
| `MainActivity.kt`         | Compose-интерфейс |

## Статус

Рабочий MVP. Поднятие/опускание гаджета и горячая смена образа реализованы
по-настоящему; имена путей и поведение USB **device-specific** - на новых
устройствах может понадобиться подстройка. Тестируй на своём железе: неправильная
работа с UDC может временно оборвать USB-соединение (лечится перезагрузкой).

Отчёты о совместимости (контроллер + SoC + результат) очень приветствуются в Issues.
